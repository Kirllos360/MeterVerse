# Sections 6-8 — Reporting, Migration & Template Architecture

---

## Section 6: Reporting Center

### Official Reporting Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| Report Engine | JasperReports 7.0.1 | All structured reports |
| Templates | JRXML | Report design (stored in database) |
| Output: Structured | PDF | Invoices, statements, official documents |
| Output: Data | Excel (.xlsx) | Tabular reports, exports |
| Output: Simple | CSV | Quick data downloads |
| Output: Dynamic | HTML (for dashboard widgets only) | Inline chart data |

### Report Types

| Category | Reports | Output Format |
|----------|---------|--------------|
| **Invoice** | Invoice PDF, Invoice Summary, Batch Invoice | PDF |
| **Customer** | Customer Statement, Aging Report, Customer List | PDF, Excel |
| **Meter** | Meter Reading History, Meter Health, Meter Inventory | PDF, Excel |
| **Reading** | Consumption Report, Reading Exception, Reading Summary | Excel, CSV |
| **Payment** | Payment Receipt, Payment Summary, Collection Report | PDF, Excel |
| **Financial** | Revenue Report, Outstanding Report, Collection Rate | PDF, Excel |
| **Operational** | Sync Status, Import History, Audit Log | Excel, CSV |
| **Regulatory** | Utility Consumption by Area, Revenue by Project | PDF, Excel |

### Report Features

| Feature | Implementation |
|---------|---------------|
| Scheduled | Daily, weekly, monthly via cron |
| Email delivery | Attach report PDF to email |
| Parameterized | Date range, project, area, customer filters |
| Versioned | JRXML templates versioned in database |
| Categories | Financial, Operational, Regulatory, Customer |
| Permissions | report:read controls visibility |

## Section 7: Migration Center

### Migration Sources

| Source | Entities | Method | Status |
|--------|----------|--------|--------|
| SBill Palm Hills | Customers, Meters, Invoices, Payments, Readings | DB-to-DB mapping | ✅ Documented |
| SBill Estates | Customers, Meters, Invoices, Payments, Readings | DB-to-DB mapping | ✅ Documented |
| Collection System (Flask) | Customers, Meters, Readings, Users | DB-to-DB mapping | ✅ Documented |
| Symbiot | Meters, Readings | API sync (active) | ✅ Active |

### Migration Pipeline

```
Source Connection
    ↓
Schema Mapping (legacy field → MeterVerse field)
    ↓
Data Extraction (batch, with pagination)
    ↓
Transformation (type conversion, status mapping)
    ↓
Validation (duplicate check, dependency check)
    ↓
Preview (show counts: total, valid, warning, error)
    ↓
Execute
    ↓
Rollback Available
```

### Migration Rules

| Rule | Implementation |
|------|---------------|
| Dry run first | Preview without committing |
| Batch size | 500 records per batch |
| Duplicate handling | Skip by unique key. Log warning. |
| Status mapping | Legacy status → MeterVerse enum (map file required) |
| Rollback | Reverse within 24 hours |
| Progress tracking | ImportRecord with per-batch status |
| Audit | Every migrated record has `migratedFrom` field |

## Section 8: Template Registry

### Template Definitions

| Entity | Template Purpose | Fields | Import/Export |
|--------|-----------------|--------|---------------|
| Customer | Bulk create/update | nameAr, nameEn, phone, email, type, projectCode, unitCode | Both |
| Meter | Bulk create/update | serialNumber, type, brand, model, projectCode | Both |
| Reading | Bulk create | meterSerial, value, date, source | Import only |
| Payment | Bulk create | customerCode, amount, method, date, invoiceNumbers | Import only |
| Invoice | — | (generated by system, not imported) | Export only |
| SIM | Bulk create/update | iccid, msisdn, provider, status | Both |
| Settlement | Bulk create | customerCode, amount, type, reason | Import only |

### Template Features

| Feature | Implementation |
|---------|---------------|
| Header row | Column names in first row |
| Example row | Second row with example data (can be deleted) |
| Validation rules | Per-column type, required, unique, enum |
| Version | Template version tracked in database |
| Download | GET /upload/template/:entityType |
| Preview | After upload, show parsed rows with status |
| Error report | Download error report with row numbers |

### Template Versioning

- Templates are versioned (v1, v2, v3) in the database
- Import validates against the template version
- Breaking changes require new template version
- Old template versions remain accessible for historical imports
